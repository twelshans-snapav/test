Halo Remote Firmware Release Notes

Version: 1.1.1

Release Date: 23-02-28

Fixes:

  • HALO-2476 — Voice control stopped working. A Room off command was required to restore voice input functionality.

  • HALO-2465 — Halo remote Control4 driver went offline after the controller restarted or was powered off/on.

  • HALO-2202 — Screen wouldn't come back on.

  • HALO-2112 — Screen was staying on.

  • HALO-2421 — Remote wasn’t going to sleep.

  • HALO-2406 — Screensaver not turning on and draining the battery.

  • HALO-2399 — If the remote was placed to/from the dock after refreshing navigators, the charging states were missing and the charging indicator was wrong.

  • HALO-2426 — Remote stuck on Adding drivers screen after FTUE, if the locale was set to a language other than English.

  • HALO-1335 — No redirection to the Watch page when it was started from Favorites.

  • HALO-2398 — Pop-up for Voice was not translated after a firmware upgrade.

  • HALO-2172 — The remote seemed to stutter every time you changed floors.

  • Pressing up and would not respond.

  • HALO-2438 — The line at the bottom of the screen (Home indicator) has been removed from all FTUE screens.

  • HALO-2459 — "Watch this space" page title in German didn’t fit the screen.

  • HALO-1790: Under My Music, the first item in album queue was not properly formatted.

  • HALO-2180 — Color buttons weren’t send programming.

  • HALO-2510 — Volume popup wasn’t displayed.

  • HALO-2418 — Sessions menu was not translated after first the refresh Navigators.

  • HALO-2405 — Subtitle for tuners remained in English after changing the locale.

  • HALO-2415 — Session Totals were one number higher than they actually were.

  • HALO-2388 — "Obtaining the list of rooms" was not translated.

  • HALO-2358 — The message shown during a refresh of Navigators was not translated.

  • Added Network Error logging to the firmware to aid in a full resolution.

Known issues:

  • QR Code/FTUE does not work when connected to a network using a 192.168.89.0/24 IP scheme.

  • If the remote stays on the Control4 boot screen, turning the remote off/on fixes the problem.

  • Press a button once to wake the remote from deep sleep mode if it is stationary and not woken up by the accelerometer. Then press the button once more to execute the command.

  • Network Error Reports: There are currently some reports of network error and project connection Issues In the field. We are resolving these Issues behind the scenes so that partners will not need to take any measures once we have fixes In place.

Pro Tip: -_Create a MAC/DHCP reservation for the primary Control4 controller and devices communicating to the controller to avoid network issues. Use OvrC or your router to do so.
