Halo Remote Firmware Release Notes

Version 1.3.2

Release Date: 23-05-09

Customer Facing Changes

  • Fixes aimed at reducing Wi-Fi dropouts that triggered the following Network Error pop-ups:

    • Wi-Fi driver stuck: In the unlikely event that recovery fails, the remote will restart automatically.

    • Wi-Fi channel change after deep sleep.

  • Decreased time to load transport controls.

Fixes

  • HALO-2661 — Remote has a black screen after waking from a deep sleep, taking ~10 seconds for the screen to recover and connect to Wi-Fi.

  • HALO-2549 — Remote gets stuck on the lock screen and the hard buttons don't respond while the Wi-Fi network is unavailable.

  • HALO-2505 — Remote got stuck on a blank screen while docked and idle.

  • HALO-2553 — The tunnel with the driver cannot be re-established and the remote does not go to sleep.

  • HALO-2662 — The remote has a black screen and is inaccessible after a firmware upgrade.

  • HALO-2701 — Remote wouldn’t charge or show the correct battery level.

  • HALO-2544 — Remote wouldn’t charge when docked.

  • HALO-2735 — Metrics for connect/disconnect events are not sent if the connected SSID name has special characters.

  • HALO-2752 — The source indicator is not always visible when selecting a source.

  • HALO-2744 — Network Error on wake up (self-recovered) 1.2.3.001076.

  • HALO-2817 — Room Off confirmation dialog doesn't appear and doesn't turn off the room (same issue with OSD button).

  • HALO-2821 — Color programming is not working.

  • HALO-2578 — Volume, Mute, Play, & Stop won't work from sleep screen.

  • HALO-2286 — Wi-Fi icon is missing even though I can connect to the remote.

  • HALO-2453 — Spinning wheel for 4-5 seconds while launching transport controls.

  • HALO-2571 — No active media icon on the sleep screen.

  • HALO-2497 — Lights/Scenes list is being refreshed each time any light is turned on/off from any endpoint.

  • HALO-2758 — During First Time User Experience (FTUE) there is no response when entering a wrong IP address and the app can crash if the Next" button is quickly pressed a few times.

  • HALO-1515 — Listen media services cannot be launched after Tuner.

  • HALO-2773 — Halo Remote Hub driver should set automatically select the default component update repository URL if the URL is not reported by the Update Manager agent.

Known Issue

  • QR code for the FTUE does not work if the network is 192.168.89.0/24.

  • When in deep sleep, you must press a button twice to wake the remote.
